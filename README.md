# 📖 Sobre o Projeto

![Azure Cloud Fundamentals](/images/modulo-1/cloud-computing.png)

O **Azure Cloud Fundamentals** reúne toda a minha jornada de aprendizado durante uma formação em **Computação em Nuvem** e **Microsoft Azure**.

Neste repositório estão organizados os materiais produzidos ao longo do curso, incluindo documentações técnicas, desafios de código e projetos práticos, registrando desde os conceitos fundamentais da computação em nuvem até o desenvolvimento de aplicações utilizando serviços da plataforma Microsoft Azure.

Além dos conteúdos da formação, o repositório também demonstra a aplicação de boas práticas de desenvolvimento e versionamento com **Git** e **GitHub**, utilizando branches, Pull Requests, Releases, Tags e documentação técnica.

Ao final da formação, este projeto tornou-se um portfólio técnico que consolida os conhecimentos adquiridos em Cloud Computing, desenvolvimento de aplicações e serviços Microsoft Azure.

---

## 🎯 Objetivos

Este repositório foi criado para reunir, de forma organizada, todo o conteúdo desenvolvido durante a formação em Microsoft Azure. Além de servir como material de consulta, ele também demonstra minha evolução prática por meio de documentações, desafios de código e projetos desenvolvidos ao longo do curso.

Os principais objetivos deste projeto são:

* ☁️ Consolidar os conhecimentos adquiridos em Computação em Nuvem e Microsoft Azure.
* 📚 Organizar toda a documentação produzida durante a formação.
* 💻 Desenvolver e documentar projetos práticos utilizando serviços da plataforma Azure.
* 🧪 Resolver desafios de código para reforçar os conceitos estudados.
* 🔄 Aplicar boas práticas de versionamento utilizando Git e GitHub.
* 🏗️ Construir um portfólio técnico que demonstre a aplicação dos conhecimentos adquiridos durante a formação.

# 🔀 Estratégia de Versionamento

Durante o desenvolvimento deste repositório foi utilizada uma estratégia simples inspirada no **Git Flow**, permitindo organizar a evolução dos estudos, controlar versões e manter um histórico consistente de todas as alterações realizadas.

---

## 🌿 Estrutura de Branches

```text
main
│
└── develop
    │
    └── feature/*
```

### Descrição das Branches

| Branch | Finalidade |
| ------- | ---------- |
| **main** | Armazena as versões estáveis e finalizadas da formação. |
| **develop** | Centraliza a integração das funcionalidades antes da publicação. |
| **feature/*** | Utilizada para o desenvolvimento de novos conteúdos, desafios, documentações e projetos. |

---

## 🔄 Fluxo de Trabalho

O fluxo de desenvolvimento adotado durante a formação seguiu as seguintes etapas:

1. Criar uma branch `feature/*` a partir da `develop`.
2. Desenvolver o conteúdo ou funcionalidade.
3. Realizar commits descritivos.
4. Enviar as alterações para o GitHub (`push`).
5. Abrir um Pull Request para a branch `develop`.
6. Realizar o merge após a revisão.
7. Criar uma **Release** ao concluir uma etapa importante da formação.
8. Criar uma **Tag** seguindo o padrão de Versionamento Semântico.
9. Realizar o merge da `develop` para a `main`, publicando uma nova versão estável do repositório.

---

## 🏷️ Versionamento Semântico

As versões deste repositório seguem o padrão de **Semantic Versioning (SemVer)**, representando a evolução da formação e dos projetos desenvolvidos.
